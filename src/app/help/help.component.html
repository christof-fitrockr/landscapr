Help & Documentation

Welcome to Landscapr. Below you will find an overview of the application's features and how to use them.

Dashboard

The Dashboard provides a central view of the application's status.

  • Status Overview: Quickly see the health and sync status of your data.
  • Synchronization: Check if your local data is in sync with remote repositories.
  • Quick Access: Jump to frequently used areas of the application.

Journeys

Journeys represent the customer or user paths through the system, mapping out their interactions and experiences.

  • List View: View all existing journeys, filter, and sort them.
  • Journey Editor: A visual tool to map out journey steps. You can link steps to underlying processes, define touchpoints, and visualize the flow.
  • Management: Create new journeys, edit existing ones, or delete them as needed.

Process

Processes define the business logic and workflows that power your organization.

  • Process List: Browse through all defined processes.
  • Process Editor: Define process steps, inputs, outputs, and logic. You can breakdown processes into subprocesses.
  • Visualizations: View processes as swimlane diagrams or flowcharts to understand dependencies and responsibilities.
  • Linking: Connect processes to API calls to show technical implementation.

Api Call

The API Call section serves as a catalog of all API definitions used in the system.

  • Catalog: List and manage API calls.
  • Grouping: Organize API calls into logical groups for better management.
  • Usage Tracking: See where API calls are used within processes and which capabilities they support.
  • Implementation: meaningful link to the Systems that implement these APIs.

Capability

Manage the business capabilities of your organization.

  • Capability Map: Visualize the hierarchy of business capabilities.
  • Linking: Connect capabilities to the systems that implement them and the processes that utilize them.
  • Strategic Planning: Use capabilities to understand what your organization does and how IT supports it.

System

The System section is an inventory of your IT landscape (Applications).

  • System Definition: Define IT systems, applications, and services.
  • API Management: List the APIs provided by each system.
  • Capability Implementation: Map systems to the business capabilities they implement.

Repositories

Manage data persistence and external synchronization.

  • Sync Settings: Configure connections to external repositories (e.g., GitHub).
  • Data Management: Control how data is saved and loaded.
